# 网页爬取到 Markdown 文档

这个 skill 帮助你从博客汇总网站爬取文章，并生成格式良好的 Markdown 文档，包含正文内容、图片、表格和 AI 总结。

## 触发条件

当用户说：
- "爬取这个网页"
- "把文章保存为 markdown"
- "从网站提取内容"
- "帮我把这个博客文章保存下来"
- 或者给了一个博客/文章 URL 并要求保存

## 工作流程

### 1. 获取网页内容

首先使用合适的工具获取网页 HTML 内容。对于简单页面可直接读取，对于动态加载的页面可能需要使用浏览器工具。

### 2. 分析页面结构

识别并提取：
- 文章标题
- 作者信息
- 发布日期
- 正文内容（主要文本段落）
- 图片（包含 alt 文本）
- 表格
- 代码块
- 引用内容

### 3. 生成总结

使用 Claude 分析文章内容，生成：
- 文章主旨概述（1-2 句话）
- 关键要点（3-5 条）

### 4. 生成 Markdown 文档

将提取的内容转换为 Markdown 格式：

```markdown
---
title: [文章标题]
author: [作者]
date: [发布日期]
source: [原始 URL]
summary: [AI 生成的文章总结]
---

# [文章标题]

> [核心要点 1]
> [核心要点 2]
> [核心要点 3]

## 正文内容

[正文内容...]

## 图片

![图片描述](图片URL)

## 总结

[AI 生成的详细总结]
```

## 输出要求

1. **文件位置**：保存在用户指定的位置，或者当前工作目录的 `scraped/` 文件夹中
2. **文件名**：使用文章标题（slug 格式）作为文件名，如 `article-title.md`
3. **图片处理**：
   - 如果是外部 URL，保留原 URL
   - 如果需要下载图片，保存到与 md 文件同名的文件夹中
4. **编码**：使用 UTF-8 编码

## 错误处理

- 如果页面无法访问：明确告知用户错误信息
- 如果内容为空或无法提取：提示用户并建议其他方法
- 如果遇到登录/付费墙：告知用户该页面需要登录

## 注意事项

- 尊重网站的 robots.txt 和使用条款
- 不要爬取需要登录的私人内容
- 对于大规模爬取，请合理控制请求频率
- 爬取后可以询问用户是否需要进行进一步处理（如翻译、摘要等）
